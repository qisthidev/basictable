About Container Breakpoint

The table on the left uses a container breakpoint of 485px.

Try resizing the browser window - the table will become responsive when its container (not the viewport) becomes too narrow.

This is useful for sidebar tables or tables within constrained layouts.

Settings Used:
new BasicTable(table, {
  containerBreakpoint: 485
});

Vanilla JS Usage Examples

HTML Structure

<table class="preline-table min-w-full divide-y divide-gray-200">
  <thead class="bg-gray-50">
    <tr>
      <th class="px-6 py-3 text-start text-xs font-medium text-gray-500 uppercase">Name</th>
      <th class="px-6 py-3 text-start text-xs font-medium text-gray-500 uppercase">Email</th>
    </tr>
  </thead>
  <tbody class="bg-white divide-y divide-gray-200">
    <tr class="hover:bg-gray-50">
      <td class="px-6 py-4 whitespace-nowrap text-sm font-medium text-gray-900">John Doe</td>
      <td class="px-6 py-4 whitespace-nowrap text-sm text-gray-500">john@example.com</td>
    </tr>
  </tbody>
</table>

Vanilla JavaScript

// Basic initialization
const table = document.querySelector('.preline-table');
new BasicTable(table);

// With custom options
new BasicTable(table, {
  breakpoint: 768,
  containerBreakpoint: 485,
  tableWrap: true
});

// Initialize Preline UI
if (typeof HSStaticMethods !== 'undefined') {

Performance Benefits

This Vanilla JS implementation offers several advantages:

  • • No jQuery dependency - Smaller bundle size and faster loading
  • • Modern JavaScript - Uses ES6+ features for better performance
  • • Tailwind CSS - Optimized CSS with automatic purging
  • • Preline UI - Accessible components with proper ARIA support

This demo showcases the Basic Table library enhanced with Preline UI components, Tailwind CSS, and pure Vanilla JavaScript.
